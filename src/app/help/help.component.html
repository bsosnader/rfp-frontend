Below you'll find detailed instructions on how to prepare your documents for uploading. Following these instructions exactly is the best way to ensure we can get useful information from your documents.

Word Documents

There are two different ways that you can choose to parse your Word (.docx) documents. The first, and preferred method is to choose questions and answers that you want to be paired together in a single entry in the database, along with filterable tags that you fill in on the Upload page.

To make questions and answers visible to the parser, follow this two step process:

  1. Highlight all text you want saved using Word's built in highlighter function with the default color.
  2. Color questions red and answers blue. Use Word's "Standard Colors," and make sure to hover over the color and ensure that it reads "Red" and "Blue," not "Dark Red" or "Light Blue."

Note: The colors must follow a pattern of red, blue, red, blue, ... It should start with red and end with blue. Don't leave any highlighted text without a color; it will be ignored.

Example:

This text will be ignored

This is question A, it will be paired with answer A.

This is answer A, it will be paired with question A.

This text will also be ignored

This is question B, it will be paired with answer B.

This is answer B, it will be paired with question B.

The second way of parsing requires no additional formatting, instead relying on the document already having styled Heading 1 and Heading 2 sections. It takes all text underneath a heading and pairs it with that heading. This method is slower and leads to a lot of junk data that is useless being extracted. It also fails if the document's headings aren't formatted with Word's "Heading 1" and "Heading 2" classes. I do not recommend using this method.

Excel Documents

The parser accepts both .xls and .xlsx documents. Similar to Word parsing, there are two methods. The first is preferred, and is also similar to the highlighting method for Word documents. Here are the steps:

  1. Using the "Fill Color" option (not the "Font Color"), select a cell containg a question and color it purple
  2. Using the "Fill Color" option again, select the cell containing the answer and color it yellow

Note: This methods assumes a few things: that a question and answer are on the same row, that there are no more than one question and answer per row, and that each question and answer is contained within a single cell.

Why yellow and purple? Yellow and purple were chosen because it is probably unlikely that a company has already used these colors as fill colors elsewhere in the document. If they do, the parser will read those cells in as well, and possibly break. You can fix it by removing the fill colors from those sections.

Example:

This is question A This will be ignored This is answer A
This is all ignored This is all ignored This is all ignored
This is question B This is answer B This is ignored

Final Note: The colors on this page may not look exactly like the ones in Word or Excel. Choose based on the exact names of the colors: red, blue, purple, yellow, not how they look precisely on this page.

The second method for parsing Excel documents simply takes the name of an entire sheet, and puts that in along with all of text from that sheet as a single entry. As you can imagine, this is slow and results in basically meaningless entries. It is not recommended to use this method.
